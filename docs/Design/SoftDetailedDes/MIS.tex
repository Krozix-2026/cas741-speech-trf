\section{Introduction}

The following document details the Module Interface Specifications for
\progname{}. This pipeline accepts different deep learning-based models for MEG analyzing, performing mTRF test for comparison.


Complementary documents include the System Requirement Specifications
and Module Guide.  The full documentation and implementation can be
found at \url{https://github.com/Krozix-2026/cas741-speech-trf}.  
The key companion documents are \href{https://github.com/Krozix-2026/cas741-speech-trf/blob/main/docs/SRS/SRS.tex}{SRS}

\noindent
The specification of \progname \ uses some derived data types: sequences,
strings, tuples, matrices, and tensors. Sequences are lists filled with
elements of the same data type. Strings are sequences of characters. Tuples
contain a list of values, potentially of different types. Matrices and tensors
denote multi-dimensional arrays of real values. In addition, \progname \ pipeline uses
functions, which are defined by the data types of their inputs and outputs.
Local functions are described by giving their type signature followed by their
specification.

\section{MIS of Audio Data Module} \label{Module_AudioData}

\subsection{Module}

Audio Data Module

\subsection{Uses}

Input Parameters Module

\subsection{Syntax}

\subsubsection{Exported Constants}

\begin{itemize}
\item $\mathcal{S}_{\mathrm{Audio}}$: the finite set of supported audio dataset identifiers available to \progname.
For the current system, this includes the training-audio datasets used for model training and the stimulus-audio dataset used in model-to-brain alignment.

\item $\mathcal{I}_{\mathrm{Audio}}$: the audio dataset index, represented
abstractly as a mapping from a dataset identifier and utterance identifier
to the corresponding audio record metadata. Each audio record contains at
least a file path, a split label (if applicable), and an utterance identifier.

\item $\mathcal{W}$: the abstract type of an audio waveform returned by this module. A value of type $\mathcal{W}$ contains the audio sample sequence together with its sampling rate.
\end{itemize}

\subsubsection{Exported Access Programs}

\begin{center}
\begin{tabular}{p{3.0cm} p{4.8cm} p{4.4cm} p{3.0cm}}
\hline
\textbf{Name} & \textbf{In} & \textbf{Out} & \textbf{Exceptions} \\
\hline
initAudioData &
audio dataset configuration: map &
initialized audio index $\mathcal{I}_{\mathrm{Audio}}$ &
unsupported dataset identifier, missing manifest, malformed metadata, file error \\

getTrainSplit &
dataset id: element of $\mathcal{S}_{\mathrm{Audio}}$ &
sequence of utterance identifiers for the training split &
dataset not initialized, training split unavailable \\

getValSplit &
dataset id: element of $\mathcal{S}_{\mathrm{Audio}}$ &
sequence of utterance identifiers for the validation split &
dataset not initialized, validation split unavailable \\

getTestSplit &
dataset id: element of $\mathcal{S}_{\mathrm{Audio}}$ &
sequence of utterance identifiers for the test split &
dataset not initialized, test split unavailable \\

getAudio &
dataset id: element of $\mathcal{S}_{\mathrm{Audio}}$, utterance id: string &
audio waveform $\in \mathcal{W}$ &
dataset not initialized, utterance not found, file error \\

\hline
\end{tabular}
\end{center}

\subsection{Semantics}

\subsubsection{State Variables}

\begin{itemize}
    \item $datasetRegistry$: a mapping from dataset names to dataset descriptors and file locations.
    \item $audioIndex$: a mapping from utterance identifiers to audio file locations and metadata.
    \item $initialized$: $\mathbb{B}$, indicates whether the module has loaded dataset structure information.
\end{itemize}

\subsubsection{Environment Variables}

\begin{itemize}
    \item Audio files stored in the external file system.
    \item Dataset directory paths provided by the Input Parameters Module.
\end{itemize}

\subsubsection{Assumptions}

\begin{itemize}
    \item The required audio datasets are present at the file locations specified by the Input Parameters Module.
    \item Each utterance identifier uniquely determines one audio sample.
    \item Audio files can be decoded into real-valued waveform sequences.
\end{itemize}

\subsubsection{Access Routine Semantics}

\noindent \textbf{initAudioData}():
\begin{itemize}
    \item transition: $datasetRegistry$, $audioIndex$, $initialized$ are populated from the configured dataset locations.
    \item exception: raised if a required dataset is unavailable, unsupported, or cannot be indexed.
\end{itemize}

\noindent \textbf{getTrainSplit}(dataset name):
\begin{itemize}
    \item output: returns $\mathcal{D}_{\mathrm{train}}$ for the requested dataset.
    \item exception: raised if the module has not been initialized or the dataset name is unknown.
\end{itemize}

\noindent \textbf{getValSplit}(dataset name):
\begin{itemize}
    \item output: returns $\mathcal{D}_{\mathrm{val}}$ for the requested dataset.
    \item exception: raised if the module has not been initialized or the dataset name is unknown.
\end{itemize}

\noindent \textbf{getTestSplit}(dataset name):
\begin{itemize}
    \item output: returns $\mathcal{D}_{\mathrm{test}}$ for the requested dataset.
    \item exception: raised if the module has not been initialized or the dataset name is unknown.
\end{itemize}

\noindent \textbf{getAudio}(utterance id):
\begin{itemize}
    \item output: returns the audio waveform $\boldsymbol{a}$ associated with the given utterance identifier.
    \item exception: raised if the utterance identifier is not present in $audioIndex$ or the file cannot be read.
\end{itemize}



\subsubsection{Local Functions}

\begin{itemize}
    \item buildAudioIndex :   
    Constructs the mapping from utterance identifiers to file paths and metadata.
    \item decodeAudio :   
    Reads an audio file and converts it to the gammatone filterbank sequence.
\end{itemize}

\section{MIS of MEG Data Module} \label{Module_MEGData}

\subsection{Module}

MEG Data Module

\subsection{Uses}

Input Parameters Module

\subsection{Syntax}

\subsubsection{Exported Constants}

\begin{itemize}
    \item $\mathcal{D}_{\mathrm{MEG}}$: the collection of MEG recordings and associated metadata available to \progname.
\end{itemize}

\subsubsection{Exported Access Programs}

\begin{center}
\begin{tabular}{p{3.2cm} p{4.2cm} p{4.2cm} p{2.4cm}}
\hline
\textbf{Name} & \textbf{In} & \textbf{Out} & \textbf{Exceptions} \\
\hline
initMEGData & - & - & unsupported dataset, file error \\
getSubjectList & - & sequence of strings & dataset not initialized \\
getMEGResponse & subject id: string, utterance id: string & $\mathbf{Y}$ & subject not found, trial not found, file error \\
getSensorCount & - & $C$ & dataset not initialized \\
getMEGSamplingRate & - & $f_s$ & dataset not initialized \\
getTrialMetadata & subject id: string, utterance id: string & tuple & subject not found, trial not found \\
getStimulusLink & subject id: string, utterance id: string & string & subject not found, trial not found \\
\hline
\end{tabular}
\end{center}

\subsection{Semantics}

\subsubsection{State Variables}

\begin{itemize}
    \item $subjectIndex$: a mapping from subject identifiers to MEG recording descriptors.
    \item $trialIndex$: a mapping from $(subject\ id,\ utterance\ id)$ pairs to dataset locations (path) and metadata (wav file for audio, and bid-format MEG data for brain signal).
    \item $sensorInfo$: metadata describing sensor identities and count $C$.
    \item $initialized$: $\mathbb{B}$, indicates whether the module has loaded the MEG dataset structure.
\end{itemize}

\subsubsection{Environment Variables}

\begin{itemize}
    \item MEG recordings and metadata stored in the external file system.
    \item Dataset directory paths provided by the Input Parameters Module.
\end{itemize}

\subsubsection{Assumptions}

\begin{itemize}
    \item The MEG dataset is available at the configured location.
    \item Each pair of subject identifier and utterance identifier uniquely identifies one MEG trial.
    \item The stimulus linkage between MEG trials and utterance identifiers is valid and consistent.
\end{itemize}

\subsubsection{Access Routine Semantics}

\noindent \textbf{initMEGData}():
\begin{itemize}
    \item transition: $subjectIndex$, $trialIndex$, $sensorInfo$, and $initialized$ are constructed from the supplied MEG dataset configuration, and $initialized := \mathrm{true}$.
    \item exception: raised if the MEG dataset is unavailable, unsupported, or cannot be indexed.
\end{itemize}

\noindent \textbf{getSubjectList}():
\begin{itemize}
    \item output: returns the sequence of subject identifiers available in $\mathcal{D}_{\mathrm{MEG}}$.
    \item exception: raised if the module has not been initialized.
\end{itemize}

\noindent \textbf{getMEGResponse}(subject id, utterance id):
\begin{itemize}
    \item output: Read the preprocessed MEG trial from the external file system, and returns the response matrix $\mathbf{Y} \in \mathbb{R}^{N \times C}$, where $N$ is the number of time samples in the trial and $C$ is the number of MEG sensors defined by $sensorInfo$.
    \item exception: raised if the subject identifier or utterance identifier is unknown, or if the corresponding recording cannot be read.
\end{itemize}

\noindent \textbf{getSensorCount}():
\begin{itemize}
    \item output: returns the number of MEG sensors $C$.
    \item exception: raised if the module has not been initialized.
\end{itemize}

\noindent \textbf{getMEGSamplingRate}():
\begin{itemize}
    \item output: returns the MEG sampling frequency $f_s$.
    \item exception: raised if the module has not been initialized.
\end{itemize}

\noindent \textbf{getTrialMetadata}(subject id, utterance id):
\begin{itemize}
    \item output: returns metadata associated with the given trial, such as timing information, recording identifiers, and preprocessing descriptors.
    \item exception: raised if the subject identifier or utterance identifier is unknown.
\end{itemize}

\noindent \textbf{getStimulusLink}(subject id, utterance id):
\begin{itemize}
    \item output: returns the stimulus identifier linked to the requested MEG trial.
    \item exception: raised if the subject identifier or utterance identifier is unknown.
\end{itemize}

\subsubsection{Local Functions}

\begin{itemize}
    \item buildSubjectIndex :   
    Constructs the mapping from subject identifiers to recording descriptors.
    \item buildTrialIndex :   
    Constructs the mapping from $(subject\ id,\ utterance\ id)$ pairs to trial locations and metadata.

\end{itemize}

    \item exception: raised if the module is not initialized.

\noindent \textbf{getPathConfig}():
\begin{itemize}

    \item exception: raised if the module is not initialized.

\section{MIS of Model Training Module} \label{Module_ModelTraining}

\subsection{Module}

Model Training Module

\subsection{Uses}

Input Parameters Module, Audio Data Module

\subsection{Syntax}

\subsubsection{Exported Constants}

None

\subsubsection{Exported Access Programs}

\begin{center}
\begin{tabular}{p{3.4cm} p{4.2cm} p{4.2cm} p{2.4cm}}
\hline
\textbf{Name} & \textbf{In} & \textbf{Out} & \textbf{Exceptions} \\
\hline
trainModel & model name: string, dataset name: string & $\boldsymbol{\theta}^{\ast}$ & training failed, invalid config \\
getParameters & model name: string & $\boldsymbol{\theta}^{\ast}$ & model not trained \\
getReport & model name: string & tuple & model not trained \\
\hline
\end{tabular}
\end{center}

\subsection{Semantics}

\subsubsection{State Variables}

\begin{itemize}
    \item $trainedModelTable$: a mapping from model names to optimized parameters $\boldsymbol{\theta}^{\ast}$.
    \item $trainingReportTable$: a mapping from model names to training summaries and diagnostics.
\end{itemize}

\subsubsection{Environment Variables}

None

\subsubsection{Assumptions}

\begin{itemize}
    \item The requested training dataset is available through the Audio Data Module.
    \item The requested model type is supported by \progname.
\end{itemize}

\subsubsection{Access Routine Semantics}

\noindent \textbf{trainModel}(model name, dataset name):
\begin{itemize}
    \item transition: the model $f_{\boldsymbol{\theta}}$ is trained on $\mathcal{D}_{\mathrm{train}}$ using the configured optimization settings. $trainedModelTable$ and $trainingReportTable$ are updated.
    \item output: returns the optimized parameters $\boldsymbol{\theta}^{\ast}$.
    \item exception: raised if configuration values are invalid, the dataset is unavailable, or optimization fails.
\end{itemize}

\noindent \textbf{getTrainedParameters}(model name):
\begin{itemize}
    \item output: returns $\boldsymbol{\theta}^{\ast}$ associated with the requested model.
    \item exception: raised if the requested model has not been trained.
\end{itemize}

\noindent \textbf{getTrainingReport}(model name):
\begin{itemize}
    \item output: returns a tuple containing training diagnostics, such as loss history and stopping information.
    \item exception: raised if the requested model has not been trained.
\end{itemize}

\subsubsection{Local Functions}

\begin{itemize}
    \item optimizeModel :   
    Optimizes model parameters using the configured training procedure.
    \item buildTrainingReport :   
    Constructs a summary of training progress and outcomes.
\end{itemize}

\noindent \textbf{getLayerCount}(model name):
\begin{itemize}
    \item output: returns the number of accessible internal layers $L$ for the requested frozen model.
    \item exception: raised if the requested model has not been loaded.
\end{itemize}

\noindent \textbf{isFrozen}(model name):
\begin{itemize}
    \item output: returns \texttt{true} if the requested model has been loaded in frozen form, and \texttt{false} otherwise.
    \item exception: raised if the requested model has not been loaded.
\end{itemize}

\subsection{Uses}

\subsection{Syntax}

\subsubsection{Exported Constants}

None

\subsubsection{Exported Access Programs}

\begin{center}

\hline
\textbf{Name} & \textbf{In} & \textbf{Out} & \textbf{Exceptions} \\
\hline
runStatAnalysis & model name: string & tuple & alignment results unavailable \\
getLayerSummary & model name: string & tuple & analysis not run \\
getBestLayer & model name: string & $\ell$ & analysis not run \\
getImprovementTable & model name: string & tuple & analysis not run \\
\hline
\end{tabular}
\end{center}

\subsection{Semantics}

\subsubsection{State Variables}

\begin{itemize}
    \item $summaryTable$: a mapping from model names to statistical summaries over layers and subjects.
    \item $bestLayerTable$: a mapping from model names to the layer index with the best evaluation score.
    \item $improvementTable$: a mapping from model names to the values of $\Delta \rho_{\ell}$ across candidate layers.
\end{itemize}

\subsubsection{Environment Variables}

None

\subsubsection{Assumptions}

\begin{itemize}
    \item Brain alignment results have already been generated for the requested model and candidate layers.
\end{itemize}

\subsubsection{Access Routine Semantics}

\noindent \textbf{runStatisticalAnalysis}(model name):
\begin{itemize}
    \item transition: retrieves alignment scores from the Brain Alignment Module, computes summaries across layers and subjects, updates $summaryTable$, $bestLayerTable$, and $improvementTable$.
    \item output: returns a tuple containing the analysis summary for the requested model.
    \item exception: raised if the required alignment results are unavailable.
\end{itemize}

\noindent \textbf{getLayerSummary}(model name):
\begin{itemize}
    \item output: returns the stored statistical summary for the requested model, including layer-wise score summaries.
    \item exception: raised if analysis has not been run for the requested model.
\end{itemize}

\noindent \textbf{getBestLayer}(model name):
\begin{itemize}
    \item output: returns the layer index $\ell$ with the best statistical outcome according to the configured selection rule.
    \item exception: raised if analysis has not been run for the requested model.
\end{itemize}

\noindent \textbf{getImprovementTable}(model name):
\begin{itemize}
    \item output: returns a tuple containing the layer-wise improvement values $\Delta \rho_{\ell}$ for the requested model.
    \item exception: raised if analysis has not been run for the requested model.
\end{itemize}

\subsubsection{Local Functions}

\begin{itemize}
    \item computeImprovement :   
    Computes the improvement over the baseline encoding score.
    \item selectBestLayer :   
    Selects the best layer according to the configured criterion.
\end{itemize}
